# Supported Game Modes

## Current Version (v1.0)

### Texas Hold'em Practice Mode

**Status**: Active  
**Description**: Single-player practice against a simulated opponent. The user plays one hand at a time and receives coaching feedback after each action.

**Key features**:
- Heads-up format (hero vs one villain)
- Scenarios generated by the Scenario Generator Agent
- Feedback on every decision
- Full hand review at the end of each hand
- Adaptive difficulty based on user level
- Memory of recurring mistakes influences scenario selection

**Supported streets**: Preflop, Flop, Turn, River  
**Supported actions**: Fold, Check, Call, Bet, Raise

## Planned Future Modes

The following modes are planned for future versions. They are **not** available in v1.

### Preflop Drills
**Description**: Focused training on preflop decision-making only.
- Starting hand ranges by position
- 3-bet/4-bet scenarios
- Blind defense situations
- Speed mode: rapid-fire preflop decisions with scoring

### Postflop Drills
**Description**: Isolated postflop scenarios without the preflop context.
- C-bet decision training
- Check-raise spots
- Bet sizing workshops
- Board texture recognition

### Bluff Detection Drills
**Description**: Teach the user to identify when an opponent is likely bluffing.
- Range analysis
- Bet sizing tells
- Action history interpretation
- Calling vs folding facing aggressive bets

### Hand Review Mode
**Description**: Import a hand history (from a real poker site) and receive analysis.
- Upload HH file
- Agent reviews all decisions
- Identifies key mistakes
- Provides improvement plan

### Tournament Simulation
**Description**: Simulates tournament-specific decisions.
- ICM concepts (chip value changes near the money)
- Push-fold situations (short stack)
- Bubble play
- Final table dynamics

### Cash Game Simulation
**Description**: Extended cash game session simulator.
- Multiple hands with stack changes
- Session-level performance tracking
- Bankroll management coaching
- Table selection concepts (educational only)

## Mode Design Principles

All modes must follow these constraints (see `context/app_constraints.md`):
1. Educational only — no real money, no gambling
2. Single player — no connection to other users
3. Coaching-first — feedback is always the goal, not just win/lose
4. Model-agnostic — any mode works with any LLM provider

---

## Adding a New Mode

To add a new game mode:
1. Define the mode in this file (name, description, key features)
2. Create a new directive in `directives/` for its scenario generation logic
3. Update the `scenario_engine.py` execution script to support the new mode
4. Update the API endpoint in `app/backend/routers/scenarios.py`
5. Update the frontend to support the new mode's UI requirements
